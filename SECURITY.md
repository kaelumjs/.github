# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.4.x   | Active    |
| < 1.4   | End of life |

## Reporting a Vulnerability

If you discover a security vulnerability in any Kaelum repository, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please email us at: **campagno.matheus@gmail.com**

Include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Suggested fix (if any)

## Response Timeline

- **Acknowledgment**: Within 48 hours
- **Initial assessment**: Within 1 week
- **Fix & release**: Depending on severity

## Disclosure Policy

- We will coordinate with you on disclosure timing
- We will credit reporters in the release notes (unless you prefer anonymity)
- We follow [responsible disclosure](https://en.wikipedia.org/wiki/Responsible_disclosure) practices
